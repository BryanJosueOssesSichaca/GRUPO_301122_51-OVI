Crear y Clonar repositorios con Git

Publicado el 06/04/2018 a las 10:13

Para empezar a utilizar git y llevar un seguimiento sobre un proyecto lo primero que se debe hacer es crear un directorio/carpeta.

Una vez creado, en la línea de comandos (GIT Bash) se utiliza el siguiente comando sobre la ruta del directorio creado:

$ git init

Este comando creará un subdirectorio llamado .git, que contiene todos los archivos necesarios para la creación del repositorio.

En caso de estar utilizando una herramienta con interfaz gráfica en lugar de la línea de comandos (GIT bash) como Github Desktop, Git Kraken o Source Tree, no es necesario digitar manualmente el comando, se accede a través de los menús de la interfaz gráfica a la creación de un nuevo repositorio.

En el caso de Github Desktop se puede editar el nombre del repositorio, agregar una descripción, seleccionar la ruta desde el computador local donde se creará el directorio que contendrá el repositorio y el subdirectorio .git, así como de manera opcional agregar un archivo README, gitIgnore y una licencia (software libre y Open Source en caso de no tener una cuenta Enterprise en GitHub) al proyecto.

El archivo gitIgnore específica a través de una serie de patrones que archivos de manera intencional GIT no debe de seguir o ignorar y el archivo README debe de facilitar una guía o descripción rápida del repositorio.

Clonar repositorios con Git

En caso de querer obtener una copia de un repositorio GIT existente, se clona y para esto, se utiliza el comando clone:

$ git clone url-del-repositorio

$ git clone https://github.com/BryanJosueOssesSichaca/GRUPO_301122_51-OVI

Para cambiar el nombre del directorio donde será clonado el repositorio se agrega un parámetro extra con el nombre del directorio deseado.

$ git clone https://github.com/BryanJosueOssesSichaca/GRUPO_301122_51-OVI Nombre-Del-Directorio

Para clonar un repositorio en GithubDesktop debe dirigirse al menú File -> clone repository, en la pestaña URL se coloca la URL del repositorio a clonar así como el directorio de destino donde se clonará el repositorio.

Obtener Información del repositorio

Estatus del repositorio (Git Bash)

La principal herramienta para comprobar en que estado estan los archivos del repositorio es el comando status:

$ git status

Devolverá información como por ejemplo la rama, si hay archivos en el staging area para hacer commit, archivos nuevos a los cuales Git no les esta haciendo un seguimiento y cambios sobre archivos existentes en el repositorio.

EJ: # On branch master nothing to commit, working directory clean

Estatus del repositorio (Github Desktop)

En el caso de Github Desktop se puede visualizar el estado de los archivos en el panel principal en la pestaña changes Github Desktop indica con un simbolo [+] los archivos nuevos que han sido agregados al repositorio, en el caso de los archivos modificados existentes un circulo amarillo y dando click sobre los cambios se visualiza en rojo las lineas de código eliminadas y en verde las lineas de código agregadas, para los archivos eliminados un simbolo [-]

Historial de commits de un repositorio (Git Bash)

Si se ha clonado un repositorio que poseia un historial de cambios, quizas se desee ver el historial de commits, para esto se utiliza el comando log

$ git log

Al ejecutar este comando se imprimirá en la pantalla de la terminal una lista que contendrá información para cada commit similar a esta:

commit 085bb3bcb608e1e8451d4b2432f8ecbe6306e7e7
Author: NombreAutor <correoElectornico@ejemplo.com>
Date: Mon Mar 17 21:52:11 2018 -0700

Descripción de los cambios del commit

Se aprecia el código comprobación SHA de cada commit, los datos del autor (Nombre y Correo electrónico), la fecha de realización y descripción del commit.

Historial de commits de un repositorio (Github Desktop)

En el panel principal al hacer click sobre la pestaña History se visualizará el historial de commits, que indica el nombre , la descripción, el nombre de la persona/contribuyente y la fecha de creación del commit asi como los archivos modificados.
