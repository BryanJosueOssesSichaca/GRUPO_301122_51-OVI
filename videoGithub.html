Como realizar Fork y Pull Request en Github

Publicado el 27/03/2018 a las 21:00

Cuando hacemos Fork de un repositorio estamos haciendo una copia exacta de ese repositorio, lo cual nos permite experimentar y realizar cambios sin afectar el repositorio original, este proceso dará como resultado dos repositorios independientes que pueden evolucionar de manera individual cada uno.
Estas copias, se utilizan normalmente para permitir a otros desarrolladores contribuir de manera segura y organizada sobre un proyecto a través de las denominadas pull-request" las cuales son peticiones que el propietario de un fork realiza al propietario del repositorio original con el fin de que haga merge (mezclar) con el repositorio original incorporando los cambios realizados en el fork.
En este vídeo, explicamos como realizar Fork sobre un repositorio, clonar ese Fork de manera local a nuestro computador utilizando GitHubDesktop y finalmente como proponer estos cambios a través de pull request en GitHub.
