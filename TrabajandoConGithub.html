Creando y sincronizando repositorio Remoto en Github

Vincular nuestro repositorio Local con nuestro repositorio remoto en Github (Git Bash)

si estamos trabajando con la linea de comandos lo primero que tenemos que hacer es crear un repositorio desde la pagina oficial de github.

hacemos click en el botón Start a project en la pagina principal de Github, a través de una interfaz web podremos crear un nuevo repositorio.

  1. Owner: la cuenta de Github a la que estará vinculado el repositorio.
  2. Repository Name: aquí debemos escribir el nombre que tendrá el repositorio.
  3. Description: una descripción opcional del respositorio.
  4. Public/private Seleccionar si nuestro repositorio será publico o privado (Para crear repositorios privados en Github debemos contar con una cuenta enterprise).
  5. Initialize this repository with a README:Crear un archivo README.md junto con el repositorio.
  6. Add gitIgnore/License: agregar licenciamiento al codigo fuente del repositorio (en caso de ser un repositorio publico, solo tendremos disponibles licencias de software libre y open source), asi como el archivo gitignore.

Creando repositorio en Github (08 de Abril del 2018) “elaboración propia”

Una vez que este creado el repositorio debemos dar click en el boton Clone or download y copiar la URL de nuestro repositorio remoto en Github.

Boton Clone or Download - Github (08 de Abril del 2018) “elaboración propia”

cuando tengamos la URL de nuestro repositorio remoto en Github debemos ir a la linea de comandos de Git (Git bash), dirigirnos a la ruta del repositorio local que hayamos creado (o crear uno nuevo, cabe mencionar que no hace falta que el repositorio local se llame igual que el remoto) e introducir el comando remote:

$ git remote add origin URL-repositorio-remoto-Github

de esta manera nuestro repositorio local estará sincronizado con nuestro repositorio remoto en Github.

Si queremos ver el repositorio remoto que tenemos vinculado debemos de pasarle el parametro -v al comando remote:

$ git remote -v

si queremos eliminar el repositorio remoto que tenemos vinculado debemos pasarle el parametro remove al comando remote

$ git remote remove origin

comando remote - Git Bash (08 de Abril del 2018) “elaboración propia”

Vincular nuestro repositorio Local con nuestro repositorio remoto en Github (Github Desktop)

una vez que hayamos creado nuestro repositorio local, solamente tendremos que dar click en el botón publish repository, se abrirá una nueva ventana donde tendremos que colocar el nombre del repositorio remoto, la descripción y podremos seleccionar si será publico o privado (Para crear repositorios privados en Github debemos contar con una cuenta enterprise).

automaticamente Github Desktop nos creará un repositorio remoto con la cuenta de github con la que hemos iniciado sesión en Github Desktop y lo sincronizará con nuestro repositorio local.

Fetch, Push y Pull

Fetch, Push y Pull (Git Bash)

Cuando nuestro repositorio local esta vinculado con un repositorio remoto, es muy importante que antes de empezar a trabajar o hacer cambios en el proyecto tengamos la ultima version con todos los commits del repositorio remoto, para esto utilizamos el comando fetch

$ git fetch: origin

el comando fetch recupera todos los datos del repositorio remoto y en caso de existir nuevos cambios, nos informará sobre estos cambios.

comando fetch - Git Bash (08 de Abril del 2018) “elaboración propia”

Es muy importante no olvidar hacer fetch antes de comenzar a realizar cambios en el repositorio local, ya que si intentamos publicar los cambios del repositorio local con una version distinta a la del repositorio remoto Git no nos lo permitirá.

comando fetch error - Git Bash (08 de Abril del 2018) “elaboración propia”

el comando fetch unicamente recupera la información pero no la une con la que tenemos en nuestro repositorio local, para actualizar nuestro repositorio local con los cambios que puedan existir en nuestro repositorio remoto debemos utilizar el comando pull

$ git pull: origin Nombre_rama

en caso tal de que la versión de nuestro repositorio local no sea la misma que la de nuestro repositorio remoto, por defecto Git no nos permitirá unir los cambios (fatal: refusing to merge unrelated histories).

para solucionar este problema tendremos que pasarle al comando pull el parametro --allow-unrelated-histories.

$ git pull: origin Nombre_rama --allow-unrelated-histories

una vez utilicemos el comando pull se abrirá el editor (Por defecto Git bash utiliza el editor vim) que hayamos configurado y en el debemos escribir la descripción del commit con la mezcla de los cambios en el repositorio remoto con el local.

comando pull - Git Bash (08 de Abril del 2018) “elaboración propia”

en este punto tendremos que empezar a realizar nuestros cambios al proyecto de manera local, cuando ya hayamos confirmados nuestros cambios a través de commits en nuestro repositorio local, tendremos que enviar esos cambios a nuestro repositorio remoto.

para enviar nuestros nuevos commits realizados en nuestro repositorio local a nuestro repositorio remoto debemos de utilizar el comando push

$ git push: origin Nombre_rama

Git nos pedira el nombre de usuario de Github y la contraseña para poder confirmar los cambios en el repositorio remoto cada vez que utilicemos el comando push, en caso de que los datos de inicio de sesión de Github sean erroneos, Git no nos permitirá mezclar nuestros cambios locales con el repositorio remoto.

comando push - Git Bash (08 de Abril del 2018) “elaboración propia”

Fetch, Push y Pull (Github Desktop)

para hacer fetch en github desktop solo tendremos que dar click en el boton Fetch Origin

Publicando Proyecto en Github Pages

la propia plataforma de Github nos ofrece un servicio de Hosting para alojar nuestros repositorios en una pagina web de una manera muy sencilla y sin necesidad de tener conocimientos en administración de servidores o clientes FTP.

este servicios es denominado Github Pages.

para poder publicar nuestro repositorio en Github pages, la rama que será publicada debe contener un archivo index.html. de lo contrario el servidor nos arrojara un mensaje de error 404 Indicando que no se encuentra el archivo index.html.

Error 404 - Github Pages (08 de Abril del 2018) “elaboración propia”

lo primero que tendremos que hacer es ir a la opción settings en la pagina de Github de nuestro repositorio remoto.

settings - github (08 de Abril del 2018) “elaboración propia”

desde allí dirigirnos al apartado Github Pages y dar click en la lista desplegable llamada Source, por defecto estara seleccionada la opcion None.

al hacer click se abrirá la lista desplegable y nos mostrara las ramas que tenemos en nuestro proyecto, en mi caso solo existe la rama Master, por lo tando dentro de Master debe existir un archivo index.html, seleccionamos la rama (en el ejemplo Master branch) y damos click en el boton save.

primero aparecera un mensaje indicando que nuestro sitio web esta listo para ser publicado, tendremos que recargar la pagina y esperar a que el mensaje cambie a Your site is published.

settings - github (08 de Abril del 2018) “elaboración propia”

URL del proyecto publicado en Github Pages

si bien Github pages nos permite modificar el dominio, por defecto la URL de un proyecto publicado en Github pages se organiza de la siguiente manera:

https://Nombre_De_Usuario_Github.github.io/Nombre_del_repositorio_Remoto/
