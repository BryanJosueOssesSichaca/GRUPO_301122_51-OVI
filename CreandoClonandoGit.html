Creando y Clonando repositorios con Git

Publicado el 06/04/2018 a las 10:13

Para empezar a utilizar git y llevar un seguimiento sobre un proyecto lo primero que debemos hacer es crear un directorio/carpeta.

Una vez haya sido creado este directorio debemos utilizar en la línea de comandos (GIT Bash) el siguiente comando sobre la ruta del directorio creado:

$ git init

Este comando creara un subdirectorio llamado .git, el cual contiene todos los archivos necesarios para la creación del repositorio.

En caso de estar utilizando una herramienta que nos bride una interfaz gráfica en lugar de la línea de comandos (GIT bash) como lo puede ser github desktop, git kraken o source tree no tendremos que digitar manualmente el comando sino acceder a través de los menús de la interfaz gráfica a la creación de un nuevo repositorio.

En el caso de Github Desktop podremos editar el nombre del repositorio, agregar una descripción, seleccionar la ruta en nuestro computador donde se creará el directorio que contendrá el repositorio y el subdirectorio .git, así como de manera opcional agregar un archivo README, gitIgnore y una licencia (software libre y open source en caso de no tener una cuenta enterprise en github) al proyecto.

El archivo gitIgnore específica a través de una serie de patrones que archivos de manera intencional GIT no debe de seguir o ignorar y el archivo README debe de facilitar una guía o descripción rápida del repositorio.

Clonando repositorios con Git

En caso de querer obtener una copia de un repositorio GIT existente necesitamos clonarlo, para clonar un repositorio necesitamos utilizar el comando clone:

$ git clone url-del-repositorio

$ git clone https://github.com/BryanJosueOssesSichaca/GRUPO_301122_51-OVI

Si queremos cambiar el nombre del directorio donde será clonado el repositorio debemos agregar un parametro extra con el nombre del directorio deseado.

$ git clone https://github.com/BryanJosueOssesSichaca/GRUPO_301122_51-OVI Nombre-Del-Directorio

Para clonar un repositorio en GithubDesktop debemos ir al menú File -> clone repository, en la pestaña URL debemos colocar la URL del repositorio a clonar así como el directorio de destino donde se clonará el repositorio.

Obteniendo Información del repositorio

estatus de nuestro repositorio (Git Bash)

la principal herramienta con la que podemos comprobar en que estado estan los archivos del repositorio es el comando status:

$ git status

nos devolvera información como en que rama nos encontramos, si hay archivos en el staging area para hacer commit, si hay archivos nuevos a los cuales Git no les esta haciendo un seguimiento y si hay cambios sobre archivos existentes en el repositorio.

EJ: # On branch master nothing to commit, working directory clean

estatus de nuestro repositorio (Github Desktop)

En el caso de Github Desktop podemos ver el estado de los archivos en el panel principal en la pestaña changes Github Desktop nos indica con un simbolo [+] los archivos nuevos que han sido agregados al repositorio, en el caso de los archivos modificados existentes un circulo amarillo y dando click sobre los cambios podremos ver en rojo las lineas de codigo eliminadas y en verde las lineas de codigo agregadas, para los archivos eliminados un simbolo [-]

Historial de commits de un repositorio (Git Bash)

Si hemos clonado un repositorio que ya poseia un historial de cambios quizas quieras ver el historial de commits, para esto debemos utilizar el comando log

$ git log

al ejecutar este comando se imprimira en la pantalla de la terminal una lista que contendra información para cada commit parecida a esta:

commit 085bb3bcb608e1e8451d4b2432f8ecbe6306e7e7
Author: NombreAutor <correoElectornico@ejemplo.com>
Date: Mon Mar 17 21:52:11 2018 -0700

Descripción de los cambios del commit

nos muestra el codigo comprobación SHA de cada commit, los datos del autor (Nombre y Correo electronico), la fecha en la que se realizo el commit y la descripción del commit.

Historial de commits de un repositorio (Github Desktop)

En el panel principal debemos hacer click sobre la pestaña History aqui podremos ver el historial de commits, el cual nos muestra el nombre del commit, la descripción del commit, el nombre de la persona/contribuyente que creo ese commit, la fecha en la cual se creo ese commit asi como los archivos modificados.
